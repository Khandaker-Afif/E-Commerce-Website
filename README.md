SwiftCart Project Technical Q&A


১) null এবং undefined এর মধ্যে পার্থক্য কী?

undefined: যখন একটি ভেরিয়েবল ডিক্লেয়ার করা হয় কিন্তু তাতে কোনো ভ্যালু অ্যাসাইন করা হয় না, তখন জাভাস্ক্রিপ্ট ডিফল্টভাবে তাকে undefined হিসেবে সেট করে। এটিও একধরনের ডেটা টাইপ।

null: নাল দিয়ে ডেভেলপাররা কোনো ভ্যালু খালি বা কিছু নেই বোঝাতে ব্যবহার করে। একে অবজেক্ট (Object) হিসেবেও বিবেচনা করা হয়।

২) JavaScript-এ map() ফাংশনের কাজ কী? এটি forEach() থেকে কীভাবে আলাদা?

map(): এটা সাধারনত একটি অ্যারের প্রতিটি এলিমেন্টের ওপর নির্দিষ্ট কাজ চালিয়ে একটি নতুন অ্যারে রিটার্ন করে কিন্তু অরিজিনাল অ্যারেকে পরিবর্তন করে না।

forEach(): এটি অ্যারের প্রতিটি এলিমেন্টের জন্য একটি ফাংশন চালায় কিন্তু কোনো কিছু রিটার্ন করে না। যা মূলত কোনো লুপ চালানোর বা সাইড-ইফেক্ট (যেমন কনসোলে দেখানো) তৈরির জন্য ব্যবহার করা হয়।

৩) == এবং === এর মধ্যে পার্থক্য কী?

== (Abstract Equality): শুধুমাত্র ভ্যালু বা মান চেক করে। যদি ডাটা টাইপ আলাদা হয়, তবে টাইপ কনভার্ট করে চেক করে। (যেমন: 5 == "5" হবে true)

=== (Strict Equality): ভ্যালু এবং ডাটা টাইপ—দুইটাই চেক করে। টাইপ আলাদা হলে false দেয়। (যেমন: 5 === "5" হবে false)

৪) API ডাটা ফেচ করার ক্ষেত্রে async/await এর গুরুত্ব কী?

API থেকে ডাটা আসতে কিছুটা সময় লাগে। জাভাস্ক্রিপ্ট সাধারণত সিনক্রোনাস বা লাইন-বাই-লাইন কাজ করে। async/await ব্যবহার করলে কোডটি ডাটা আসা পর্যন্ত অপেক্ষা করে এবং ডাটা আসার পর পরবর্তী লাইনে যায়। এর ফলে কোড পড়তে সুবিধা হয় এবং Callback Hell বা জটিল প্রমিস স্ট্রাকচার এড়ানো যায়।

৫) JavaScript-এ Scope (Global, Function, Block) এর ধারণাটি ব্যাখ্যা করো।

Global Scope: যদি কোনো ভেরিয়েবল কোনো ফাংশনের বাইরে ডিক্লেয়ার করা হয়, তবে সেটি পুরো কোডের যেকোনো জায়গা থেকে এক্সেস করা যায়।

Function Scope: ফাংশনের ভেতরে ডিক্লেয়ার করা ভেরিয়েবল শুধুমাত্র ওই ফাংশনের ভেতরেই কাজ করে।

Block Scope: let বা const দিয়ে কোনো থার্ড ব্র্যাকেট {} বা ব্লকের ভেতরে (যেমন if বা for লুপ) ডিক্লেয়ার করা ভেরিয়েবল শুধুমাত্র ওই ব্লকের ভেতরেই সীমাবদ্ধ থাকে।
